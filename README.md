# ByteBites Tinker Activity

This project models the backend logic for a simple food ordering app called ByteBites using Python classes and basic algorithms. The main concept students needed to understand was how to break a vague feature request into a few clear objects with specific responsibilities. Students are most likely to struggle with identifying the correct classes, deciding where methods belong, and keeping the design simple instead of adding extra features. AI was helpful for generating first drafts of UML diagrams, class scaffolds, and test ideas, especially when given the project spec as context. However, AI could also be misleading when it introduced unnecessary complexity, extra classes, or methods that were not supported by the feature request. One way I would guide a student without giving away the answer is by asking them to point to the exact sentence in the spec that justifies each class, attribute, or method they want to add. This helps students use the requirements as evidence and practice verifying AI-generated suggestions instead of accepting them blindly.
